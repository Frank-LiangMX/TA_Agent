# 决策：单 Agent + 多工具架构

> 日期：2026-05-10 | 状态：已采纳

## 背景

TA Agent 需要处理资产扫描、FBX 解析、贴图检查、AI 推断、审核、入库等多个环节。技术选型面临单 Agent 与多智能体两种选择。

## 决策

选择**单 Agent + 多工具**架构，而非多智能体框架（如 CrewAI、AutoGen）。

## 理由

1. **流程线性**：TA 工作流本质是线性串联（检查→修复→配置→导出），不需要多 Agent 并行
2. **知识统一**：核心知识是同一套 TA 领域知识，不需要不同专业 Agent
3. **维护性**：单 Agent 更好维护、好调试、出错时定位更快
4. **成本**：少一次 LLM 调用就少一份延迟和费用

## 何时升级

当需要并行处理多个资产、或需要专业分工（shader 分析 vs 模型拓扑分析）时，再考虑多智能体。
