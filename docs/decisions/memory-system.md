# 决策：三层项目记忆系统

> 日期：2026-05-10 | 状态：已采纳

## 背景

Agent 的 AI 推断层是无状态的，用户纠正过的错误下次还会犯。需要让 Agent 具备跨会话的学习能力。

## 决策

采用**三层记忆结构**：L0 项目画像 + L1 纠正规则 + L2 归档。

## 设计原则

1. **无纠正不记忆**：只有用户显式纠正才写入，LLM 自己的推测永远不存
2. **最小充分**：只编码精简规则，不存原始对话
3. **按需注入**：根据当前资产特征匹配相关规则，不全量塞入 prompt
4. **自压缩**：超过阈值自动合并，token 消耗始终可控

## 记忆生命周期

```
用户纠正 → L2 归档 → 累积 > 10 条
  → LLM 合并相似纠正 → L1 规则
  → 规则被多次命中且未被纠正 → 置信度上升
  → 长期不命中 → 降级淘汰
  → 可靠规则 → 提炼为 L0 项目画像
```

## 接口

通过依赖注入与主流程解耦，可选实现：`FileMemoryProvider`（本地）、`NullMemoryProvider`（测试）、`DBMemoryProvider`（未来团队版）。
