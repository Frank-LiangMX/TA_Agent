# 分布式架构决策：会话本地存储 + 服务器数据共享

> 日期：2026-05-20
> 状态：已确定

---

## 背景

原架构为单机模式，所有数据存储在用户本机。现需要支持多用户协作，将部分数据存储在公司服务器。

## 决策

### 数据存储策略

| 数据类型 | 存储位置 | 原因 |
|---------|---------|------|
| 会话记录 | 用户本机 | 包含对话历史、个人操作习惯，属于隐私数据 |
| 资产数据库 | 服务器 | 团队共享，需要集中管理 |
| 审核记录 | 服务器 | 团队协作，需要共享审核状态 |
| 项目配置 | 服务器 | 团队统一配置 |
| 记忆系统 | 用户本机 | 个人学习记录，属于隐私数据 |

### 架构图

```
用户本机（Electron）                 公司服务器
┌─────────────────────────┐        ┌─────────────────────────┐
│ Electron 桌面应用        │        │ server.py               │
│ ├── 会话记录（本地）     │        │ ├── 资产数据库          │
│ ├── 记忆系统（本地）     │        │ ├── 审核记录            │
│ ├── Agent（本地执行）    │        │ ├── 项目配置            │
│ ├── Blender（本地）      │ ─────► │ └── Web UI（查看用）    │
│ └── 本地资产文件         │ 同步   │                         │
└─────────────────────────┘        └─────────────────────────┘

数据流：
1. 本地分析资产 → 结果存本地会话
2. 同步分析结果 → 服务器资产数据库
3. 审核操作 → 同步到服务器
```

### 用户配置流程

```
首次启动 Electron：
1. 输入服务器地址（如 10.11.131.124）
2. 输入用户名（如 张三）
3. 保存配置到本地 userData 目录

后续启动：
1. 自动读取配置
2. 连接服务器
3. 正常使用
```

### 服务器职责

```
服务器只负责：
1. 存储资产数据库（团队共享）
2. 存储审核记录（团队共享）
3. 存储项目配置（团队统一）
4. 提供 Web UI（查看统计）

服务器不负责：
1. 不存储会话记录
2. 不存储记忆系统
3. 不执行 Agent（用户本机执行）
4. 不执行 Blender（用户本机执行）
```

## 原因

### 为什么会话存本地

1. **隐私性**：会话包含用户的操作历史、提问内容，属于个人隐私
2. **安全性**：本地存储，不会泄露给其他用户
3. **性能**：本地读写快，不依赖网络
4. **离线可用**：断网时仍可查看历史会话

### 为什么资产数据存服务器

1. **团队共享**：资产分析结果需要团队成员共同查看
2. **集中管理**：统一的资产数据库，避免数据分散
3. **审核协作**：审核记录需要多人协作
4. **Web UI**：提供浏览器访问，不需要安装客户端

## 影响

### 需要改动的文件

| 文件 | 改动 | 说明 |
|------|------|------|
| `electron/main.js` | 添加首次配置界面 | 输入服务器地址、用户名 |
| `session_manager.py` | 存储路径改为本地 | 使用 Electron userData 目录 |
| `server.py` | 去掉会话相关 API | 会话不再存服务器 |
| `server.py` | 加强资产数据 API | 分析结果同步 |

### 用户体验

```
首次启动：
├── 输入服务器地址
├── 输入用户名
└── 保存配置

日常使用：
├── 本地分析资产
├── 结果自动同步到服务器
└── 其他用户可通过 Web UI 查看
```
