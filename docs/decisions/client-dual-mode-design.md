# 客户端双模式设计

> 日期：2026-05-21
> 状态：已确定

---

## 一、背景

TAgent 客户端需要支持两种使用场景：
- **本地模式**：独立使用，自己配置 LLM API，数据存储在本地
- **联机模式**：连接公司服务器，多人协作，数据共享

## 二、模式对比

| 功能 | 本地模式 | 联机模式 |
|------|---------|---------|
| LLM 调用 | 用自己的 API Key | 用服务器配额 |
| 资产数据 | 存本地 | 同步到服务器 |
| 审核记录 | 存本地 | 同步到服务器 |
| 项目配置 | 本地配置 | 从服务器获取 |
| 记忆系统 | 本地学习 | 服务器共享 |
| 会话记录 | 存本地 | 存本地 |
| 多人协作 | 不支持 | 支持 |

## 四、数据存储策略

### 4.1 本地模式

```
本地存储：
├── .ta_agent/sessions/     # 会话记录
├── .ta_agent/memory/       # 记忆系统
├── .ta_agent/configs/      # 项目配置
└── tag_store/              # 资产数据库
```

### 4.2 联机模式

```
本地存储：
└── .ta_agent/sessions/     # 会话记录（始终本地）

服务器存储：
├── 资产数据库
├── 审核记录
├── 项目配置
└── 记忆规则
```

## 五、模式切换设计

### 5.1 切换规则

```
切换到联机模式：
├── 会话记录：保留本地
├── 资产数据：开始同步到服务器
├── 项目配置：从服务器获取
└── 记忆系统：从服务器获取

切换到本地模式：
├── 会话记录：保留本地
├── 资产数据：使用本地数据库
├── 项目配置：使用本地配置
└── 记忆系统：使用本地记忆
```

### 5.2 数据隔离

```
关键设计：
├── 会话记录始终存本地（两种模式都一样）
├── 资产数据按模式分开存储
│   ├── 本地模式 → 本地 SQLite
│   └── 联机模式 → 服务器
├── 切换时数据不自动同步
└── 用户手动选择是否同步
```

### 5.3 手动同步

```
联机模式下：
┌─────────────────────────────────────┐
│  数据同步                           │
│                                      │
│  [上传本地数据到服务器]              │
│  [下载服务器数据到本地]              │
└─────────────────────────────────────┘
```

## 八、LLM 调用逻辑

```python
def call_llm(messages):
    if config.mode == "local":
        # 本地模式：用用户自己的 API Key
        api_key = config.local.llm_api_key
        model = config.local.llm_model
    else:
        # 联机模式：用服务器配额
        # 先检查是否超限
        check_result = check_usage_limit(config.online.user_id)
        if not check_result["allowed"]:
            raise Exception("调用次数已达上限")

        api_key = get_server_api_key()  # 从服务器获取
        model = get_server_model()

    return call_api(api_key, model, messages)
```
