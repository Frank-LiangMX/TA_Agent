# 工作流程

> 从资产制作到入库的完整链路，以及多用户协作流程设计

---

## 一、单人工作流（本地模式）

### 完整流程

```
美术做资产（Maya/Blender/3ds Max）
    ↓
保存到本机目录（如 D:\Assets\Characters\）
    ↓
打开 TAgent（双击 exe 或 python agent.py）
    ↓
"分析 D:\Assets\Characters"
    ↓
Agent 自动执行：
  ├── 扫描目录（os.walk）
  ├── 解析 FBX（Blender subprocess）
  ├── 检查贴图（Pillow）
  ├── 命名检查（regex）
  ├── 面数检查（数值比较）
  ├── AI 推断（LLM 分类/材质/风格）
  └── 生成缩略图
    ↓
查看分析结果 → 修正问题 → 审核通过 → 入库 → 在 UE5 中运行导入脚本 → SVN commit
```

### 涉及组件

| 步骤 | 组件 |
|------|------|
| 扫描 | `tools/file_info.py` |
| FBX 解析 | `tools/mesh_fbx.py` + Blender |
| 贴图检查 | `tools/texture.py` |
| 命名检查 | `tools/naming.py` |
| AI 推断 | `tags/inferrer.py` |
| 审核 | `tools/review.py` |
| 入库 | `tools/intake.py` |

## 二、团队工作流（中心模式）

### 架构

```
美术 A 的电脑                    中心服务器
├── TAgent（本地）                ├── TAgent（中心）
├── SVN 工作副本                  ├── 共享数据库
├── Blender                      ├── 规范文档
├── 资产文件                      └── Web 前端
└── UE 引擎
```

### 数据同步

```
本机 Agent                          中心服务器
    ├─ 分析完成 ──POST /api/sync──────►│
    ├─ 用户修正 ──POST /api/corrections►│
    │◄─ 拉取规范 ──GET /api/conventions─┤
    │◄─ 审核结果 ──GET /api/reviews────┤
```

## 三、SVN 集成

### 双目录方案

```
SVN 仓库
├── ArtResources/          ← 原始资产（FBX、贴图）
└── Content/               ← UE 导入资产 (.uasset)
```

两个目录子路径一一对应。入库时 Agent 复制原始文件到 ArtResources/，UE 导入后生成 .uasset 到 Content/。

### 与 SVN 的关系

Agent 是 SVN 提交前的"质量门禁"，不替代 SVN。文件版本管理由 SVN 负责，质量检查/命名规范/入库自动化由 Agent 负责。

### 推荐策略：混合模式

```
ArtResources/ 提交 → 自动分析，不阻碍（日常）
Content/ 提交 → 必须审核通过才能提交（关键节点）
```

先用被动监控不强制，等团队习惯后再加 pre-commit hook。

## 四、知识沉淀

### 记忆层级

| 层级 | 内容 | 存储 | 更新方式 |
|------|------|------|----------|
| L0 项目画像 | 资产分布、命名规律 | 本机+中心 | 自动统计 |
| L1 推断规则 | 从修正中提炼的规则 | 本机+中心 | 累积 10 条自动压缩 |
| L2 修正记录 | 用户纠正的原始数据 | 本机+中心 | 每次纠正记录 |

### 规则来源流程

```
用户说 "@开头的是动画文件"
    → Agent 调用 add_custom_rule(pattern="@*.*", type="animation")
    → 写入项目配置文件
    → 下次分析自动生效
```

## 五、团队推广策略

### 核心原则

**帮美术省事，不是给美术添堵。**

| 错误做法 | 正确做法 |
|---------|---------|
| "你的命名不规范" → 挑刺 | "我帮你一键重命名 20 个文件" → 省事 |
| "必须通过审核才能提交" → 强制 | "你先提交，有问题我告诉你" → 无感 |
| "配置 config.py" → 麻烦 | "双击图标，拖文件夹就行" → 零门槛 |

### 推广路径

```
第 1 周：TA 自用，熟悉工具
第 2 周：TA 帮美术用，"你把文件夹发给我"
第 3 周：美术自己用，"你自己拖文件夹进去，比发给我快"
第 4 周：变成习惯，"做完资产顺便跑一下 Agent"
```

### 最容易被接受的功能

一键重命名、批量导入 UE、缩略图预览、贴图格式检查。
