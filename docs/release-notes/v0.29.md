# v0.29 (2026-06-05)

## 新功能

### SubAgent 工具事件流

后端 `agent_loop` 现在在子 agent 调用工具时 emit `subagent_tool` 事件，工具执行完成后 emit `subagent_progress` 事件。WebSocket 推送到前端，前端 SubAgentCard 实时更新。

事件流：`agent_loop` → `progress_hook.emit_subagent_*` → `server._forward_subagent_events` → WebSocket → `subagent-events.ts` → jotai store → SubAgentCard UI

效果：用户在 general 模式下发"用 explorer 看一下 src 目录"时，**能在 SubAgentCard 折叠行里实时看到子 agent 调过了哪些工具**，不再是提交后看到完整结果的黑盒。

### SubAgentCard 视觉对齐 Proma

v0.28 的 SubAgentCard 是**独立卡片**（突兀），v0.29 改成**单行折叠按钮**（对齐 Proma 折叠行风格）：

- **折叠态**：状态图标 + 子 agent 角色图标 + 语义短语（"正在探索代码结构"）+ 工具调用计数 + 后台标识
- **展开态**：
  - 提示词（可单独折叠）
  - 子工具列表（每行一个工具 + 参数预览）
  - 最终输出（**Markdown 渲染**）
  - 底部统计行（步数 / 耗时 / tokens）
- 默认折叠，不占空间

## 改动

- `apps/web/src/components/agent/SubAgentCard.tsx` — 完全重写为折叠行风格
- `apps/web/src/lib/subagent-phrase.ts` — 新增工具短语映射
- `apps/web/server/progress_hook.py` — 新增 `emit_subagent_event` / `emit_subagent_tool` / `emit_subagent_progress`
- `apps/web/server/server.py` — 新增 `_forward_subagent_events`，在工具轮询循环里被调用
- `backend/agent_main.py` — `agent_loop` 新增 `subagent_context` 参数，工具调用前后 emit 事件
- `packages/tools/agent_tool.py` — `SubAgentOrchestrator` 构造 `subagent_context` 传给 `agent_loop`
