# v0.28 (2026-06-05)

## 新功能：通用模式 SubAgent

主 agent 现在可以在 general 模式下把任务委派给"专业子角色"（`explorer` / `researcher` / `code-reviewer`），子角色用独立上下文、独立工具白名单、独立模型执行任务。

### 3 个内置 SubAgent

| 名字 | 用途 | 默认模型 tier |
|------|------|--------------|
| `explorer` | 代码探索（只读：扫目录、看文件、调用关系） | haiku（轻量） |
| `researcher` | 技术调研（第三方库、API、最佳实践） | haiku（轻量） |
| `code-reviewer` | 代码评审（质量、bug、风格） | sonnet（主模型） |

### 3 个新工具（仅 general 模式）

- **`Agent(subagent_type, prompt, description, run_in_background=False)`** — 委派任务
- **`TaskOutput(task_id, block=True, max_wait_ms=30000)`** — 拿后台任务结果
- **`TaskStop(task_id)`** — 取消后台任务

### 同步 / 后台两种模式

- 同步：阻塞等待，结果直接显示
- 后台：立即返回 `task_id`，主 agent 继续响应其他消息

### 前端 SubAgentCard

聊天气泡里显示 SubAgent 任务卡片：
- 4 种状态：running / completed / error / stopped
- 嵌套工具调用默认折叠
- 后台任务点 [查看进度] 弹侧栏详情

### 设置：per-subagent 模型覆盖

设置 → Agent → "SubAgent 模型覆盖" → 可单独覆盖任一子 agent 的模型。

### 安全约束

- 子 agent 工具白名单**不含** `Agent` / `TaskOutput` / `TaskStop`（防递归）
- TA 模式**不**暴露 SubAgent 工具（保持单 agent）
- `subagent_runs.jsonl` 记录每次运行（模型、步数、耗时、错误）

## 修复

- `get_subagent_model` 修复：使用 `get_active_provider_model()` 解析 fallback，适配新 `app-config.json` 的 `providers[]` 结构（日志里现在能正确记录实际模型名）
- ModelSelector 修复：模型列表为空时不再显示"暂无模型"静态标签，仍可点开 Dialog（设置页添加模型后无需重启即可在会话中选用）
- Electron 渲染进程修复：`main.tsx` 顶部兜底设置 `window.$RefreshReg$`，解决 vite @react-refresh preamble 时序问题（dev electron 不再白屏）
- 父级取消级联：`progress_hook.cancel_session()` 现在用 `list()` 拷贝避免迭代时修改 dict

## 优化

- 真实 LLM 端到端测试套件：8 个测试场景（同步 3 类型 / 后台立即返回 / 并发 / via tool function / 日志 / 路由），需 `RUN_LLM_E2E=1` 才跑
- Mock 端到端测试：24 个测试覆盖同步 / 后台 / schema / 模式隔离 / 防递归 / 模型路由 / 日志 / 错误处理 / 取消级联
