## 问题清单

### 核心痛点

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| 手动启动 Server | 🔴 致命 | 美术需在 UE5 控制台执行 Python 命令 |
| UE5.7 兼容性错误 | 🔴 严重 | `AssetImportTask.Result` 属性被保护 |

### 体验问题

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| 无状态反馈 | 🟠 中等 | 等待 120 秒无中间状态 |
| 路径规划不合理 | 🟠 中等 | 目标路径由调用者传入，无验证 |
| 导入设置不智能 | 🟡 一般 | 只有 3 个基础选项 |

### 工程问题

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| 路径硬编码 | 🟡 一般 | `F:/ta_agent` 写死 |
| 无重试机制 | 🟡 一般 | 导入失败后无自动重试 |
| 线程安全 | 🟡 一般 | 子线程调用 UE5 API |
| 无健康检查 | 🟢 轻微 | 导入前不检查 Server 状态 |
| 错误信息不友好 | 🟢 轻微 | 英文错误，无解决方案 |

## 改进方案

### 方案对比

| 方案 | 解决问题 | 开发周期 | 推荐度 |
|------|----------|----------|--------|
| A: UE5 插件自动启动 | 手动启动问题 | 3-5 天 | ⭐⭐⭐⭐⭐ |
| B: 外部启动器 | 手动启动问题 | 1-2 天 | ⭐⭐⭐⭐ |
| C: 修复兼容性 + 优化体验 | 其余全部 | 2-3 天 | ⭐⭐⭐⭐⭐ |

**推荐策略**：方案 A + 方案 C 组合实施。

### 方案 A：UE5 插件

开发 `TAAssetBridge` 插件，Editor 启动时自动加载 Server：

```
UE5 启动 → 插件自动加载（延迟 5 秒）→ Server 自动启动 → 美术无需操作
```

### 方案 C：兼容性修复

UE5.7 中 `AssetImportTask.Result` 被保护，改为检查目标路径是否有资产：

```python
if "Result" in error_str and "protected" in error_str:
    assets = asset_registry.get_assets_by_path(target_dir)
    if assets:  # 资产已导入成功
        return {"success": True, "warning": "UE5.7 兼容性警告"}
```
